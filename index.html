Stacks

만들고자 하는 목표에 대해 필요한 기술들을 빠르게 습득하고 구현합니다. 할 수 있는 경계선을 더 넓혀 나가고자 합니다.

기술 구분 로고 활용 수준 주관적 평가
JavaScript 해당 프로그램에 대한 다양한 연관 키워드를 풍부하게 넣어 어느정도의 수준으로 활용하고 있는지를 적어주시면 됩니다.
① JavaScript의 경우 연관 키워드는 ECMAScript, Webpack, Babel, TypeScript 등의 키워드를 넣어서
Node.js 해당 프로그램에 대한 다양한 연관키워드를 풍부하게 넣어 어느정도의 수준으로 활용하고 있는지를 적어주시면 됩니다.
② 자신의 기술적인 능력을 적어주시면 보시는분의 빠른 판단에 도움이 됩니다.
React 해당 프로그램에 대한 다양한 연관키워드를 풍부하게 넣어 어느정도의 수준으로 활용하고 있는지를 적어주시면 됩니다.
③ 연관 키워드는 strong태그로 감싸주시면 SEO 검색 최적화에 더욱 도움이 됩니다.
Express 해당 프로그램에 대한 다양한 연관키워드를 풍부하게 넣어 어느정도의 수준으로 활용하고 있는지를 적어주시면 됩니다.
Koa 해당 프로그램에 대한 다양한 연관키워드를 풍부하게 넣어 어느정도의 수준으로 활용하고 있는지를 적어주시면 됩니다.
MongoDB 해당 프로그램에 대한 다양한 연관키워드를 풍부하게 넣어 어느정도의 수준으로 활용하고 있는지를 적어주시면 됩니다.
AWS EC2 해당 프로그램에 대한 다양한 연관키워드를 풍부하게 넣어 어느정도의 수준으로 활용하고 있는지를 적어주시면 됩니다.
AWS S3 해당 프로그램에 대한 다양한 연관키워드를 풍부하게 넣어 어느정도의 수준으로 활용하고 있는지를 적어주시면 됩니다.

FrontEnd

JavaScript를 가장 많이 사용해 왔으며 가장 자신있는 언어입니다. 특히 React에 관심이 많습니다.

Backend

Node.js를 가장 많이 사용하며 Express.js, Koa 등의 프레임워크를 활용하고 있습니다 Deno를 공부해보고 싶습니다.

Database

MongoDB를 가장 좋아하며 Mongoose를 활용하고 있습니다. 물론 MySQL도 사용하는 중이며 GraphQL에 관심이 많습니다.

Utilities

GraphQL, Asana, Trello, AdobeXd, GitHub, Redmine 등 사용하고 있습니다. 더 많이 알 수록 효율은 더 높아진다고 믿습니다.
